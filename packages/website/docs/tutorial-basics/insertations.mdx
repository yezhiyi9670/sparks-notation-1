# 换气记号、小括号、层叠连缀 {#insertations}

简谱中还有几个写在音符之间（而不是音符上）的符号，它们是换气记号和小括号。另外，Sparks NMN 还引入了一个“层叠连缀”记号，因为这个现象在流行歌曲中并不少见。它们在 Sparks NMN 中的记号是这些：

<Cr.InsertationReference />

这些符号只是简单地均匀分布到两个音符之间。不像歌词中的连接标点，目前自动布局算法不会考虑插入记号所占据的位置。

## 用于可去间奏的小括号 {#parens}

如果乐谱只记录了歌曲的主旋律，没有将伴奏分到另一个声部，那么间奏部分有时会被加上小括号。Sparks NMN 也提供了添加小括号的方式。只需要在音符序列中插入 `&lpr;` 和 `&rpr;` 分别表示左括号和右括号的位置即可。例如歌曲《数鸭子》中的前奏：

<C.RealtimeScore efRange={[1,5]} children={
`P: 1=C 4/4
Rp: font_lyrics=Roman,CommonSerif/600/0.95
===
N: X X (XX) X | (XX) (XX) X 0 | (XX) (XX) (XX) X | (XX) (XX) X 0 |
Lc[1.]: 门前大桥下，游过一群鸭。快来快来数一数，二四六七八。
---
// highlight-next-line
N: &lpr; (1e.(1e)) (55) (36) (53) | (21) (23) 1 - &rpr; |
---
N: ||: 3 1 (33) 1 | (33) (56) 5 - | (66) (65) (44) 4 | (23) (21) 2 - |
Lc[1.]: 门前大桥下，游过一群鸭。快来快来数一数，二四六七八。
Lc[2.]: 赶鸭老爷爷，胡子白花花。唱呀唱着家乡戏，还会说笑话。
`
} />

插入记号也可以用在表示减时线的小括号内。它们是独立的记号，并不依附于其前面或后面的音符。

这里要特别提醒一点：小括号只应当用于“可去”的间奏。如果这部分间奏包含了一些关键的变速、转调或情感变化等，使得去除间奏后歌曲变得不再连贯，甚至是非常别扭，则无论如何不应该为其添加小括号。

## 换气记号 {#interval}

换气记号用于两个音符相连的地方，表示在演唱/弹奏时，此处可以通过减少前一个音符的时长，以空出一小段时间进行换气或抬手。换气记号的写法是 `&int;`。

<C.RealtimeScore efRange={1} children={
`P: 1=C 4/4
// highlight-next-line
N: (32)(35)(05)(52) | (17d)(12)3&int; (6d5d) | (6d6)(54)(35) 5 | (43)(21) 2&int; (32) |
`
} />

如果换气或抬手的地方已经有一个休止符，则没有必要再使用换气记号。

## 层叠连缀 {#cascade}

层叠连缀是流行音乐中一种常见的现象，即两个紧贴着的音符，后者开始时前者还没有完全结束，构成短暂的多声部关系。Sparks NMN 为此定义了专门的记号，写作 `&cas;`。

下面的片段来自歌曲[《芝士配曲奇》](https://www.kuwo.cn/play_detail/162604893)。

<C.RealtimeScore efRange={1} children={
`P: 1=? 4/4
Rp: font_lyrics=Roman,CommonSerif/600/0.95
====
// highlight-next-line
N: (56)(03)(53)(15d) | (6d1)(51)(2&cas;5d)(12) | 3(31)(25)(51) | (17d)(15)3 (05) |
Lc: 道理，不想逃离，奶油加进甜心。搅拌机混入了巧克力，想念藏进杯底，越
`
} />

用多声部的写法，这大致等价于，

<C.RealtimeScore efRange={1} children={
`P: 1=? 4/4
Rp: font_lyrics=Roman,CommonSerif/600/0.95
====
// highlight-next-line
N["V"_"1"]: (56)(03)(53)(15d) | (6d1)(51)2 0 |
Lc: 道理，不想逃离，奶油加进甜心。
// highlight-next-line
N["V"_"2"]: e / - - (05d)(12) | 3(31)(25)(51) | (17d)(15)3 (05) |
Lc: 搅拌机混入了巧克力，想念藏进杯底，越
`
} />

不过用多声部写这种东西，确实是有点大材小用了吧。
