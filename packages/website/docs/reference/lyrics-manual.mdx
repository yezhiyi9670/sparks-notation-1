## 用斜杠划分 {#dividing-with-slashes}

歌词中的音节和连接标点，用斜杠 `/` 隔开。

两条斜杠之间留空，将形成一个占位符，占据掉一个可配对音符的位置。

:::caution 常见错误

不要在歌词行开头处添加一个多余的斜杠。这会导致开头出现一个占位符。

:::

## 音节或连接标点 {#lyric-char}

### 基本格式 {#format-basic}

音符或连接标点具有这样的格式：`<#?><@?><本体><_?>`。

- 前缀 `#`，表示此音节不与音符配对，而是均匀分布在两侧音节之间。也就是说使这个音节成为连接标点。
- 前缀 `@`，表示添加下划线。
- 后缀 `_`，表示在此音节之后绘制一条与相应音符等长的下划线。

:::caution 常见错误

目前前缀 `#` 和 `@` 的顺序不可以颠倒。但是真的有人把它们会一起用吗？

:::

### 本体格式 {#format-self}

本体可以是这样的格式：

- 直接字面写出的歌词内容，表示此音节内容，且音节不带前后置标点。
- 一个带双引号的字符串，形如 `"A"`，表示此音节内容，且音节不带前后置标点。
- 两个紧邻的字符串，形如 `"意""，"`，第一个表示音节内容，第二个表示后置标点。
- 三个紧邻的字符串，分别表示前置标点、音节内容和后置标点。
